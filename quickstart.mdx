This guide will walk you through setting up the plugin and generating your first JOOQ classes from Flyway migrations.

## Step 1: Apply the plugin

Add the plugin to your `build.gradle.kts`:

```kotlin build.gradle.kts
plugins {
    id("io.github.usmanzaheer1995.flyway-jooq-convention-plugin") version "1.0.9"
}
```

## Step 2: Configure the plugin

Add the `jooqConventions` configuration block to your `build.gradle.kts`:

```kotlin build.gradle.kts
jooqConventions {
    // Package where JOOQ code will be generated
    targetPackage = "com.example.jooq.generated"
    
    // PostgreSQL version for Testcontainer (optional)
    postgresVersion = "16-alpine"
    
    // Database credentials for temporary container (optional)
    databaseName = "testdb"
    username = "test"
    password = "test"
    
    // Schema to generate code for (optional)
    inputSchema = "public"
    
    // Tables to exclude from generation (optional)
    excludedTables = "flyway_schema_history"
}
```

<Note>
  All configuration properties are optional. If you don't provide a `jooqConventions` block, sensible defaults will be used.
</Note>

### Configuration defaults

If you omit any properties, the following defaults apply:

| Property | Default Value |
| :--- | :--- |
| `targetPackage` | `com.example.jooq.generated` |
| `postgresVersion` | `16-alpine` |
| `databaseName` | `testdb` |
| `username` | `test` |
| `password` | `test` |
| `inputSchema` | `public` |
| `excludedTables` | `flyway_schema_history` |

## Step 3: Create a migration file

Create your first Flyway migration in the standard Flyway directory structure:

```text
src/main/resources/db/migration/
  └── V1__Initial_schema.sql
```

Here's a sample migration that creates a users table:

```sql V1__Initial_schema.sql
CREATE TABLE users (
    id BIGSERIAL PRIMARY KEY,
    username VARCHAR(255) NOT NULL UNIQUE,
    email VARCHAR(255) NOT NULL UNIQUE,
    created_at TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_users_username ON users(username);
CREATE INDEX idx_users_email ON users(email);
```

<Note>
  Flyway migration files must follow the naming convention: `V{version}__{description}.sql`
  
  For example:
  - `V1__Initial_schema.sql`
  - `V2__Add_users_table.sql`
  - `V3__Add_orders.sql`
</Note>

## Step 4: Generate JOOQ classes

Run the JOOQ generation task:

```bash
./gradlew generateJooq
```

This command will:

<Steps>
  <Step title="Start PostgreSQL container">
    Spin up a temporary PostgreSQL container using Testcontainers
  </Step>
  
  <Step title="Run migrations">
    Apply all migrations from `src/main/resources/db/migration` to the container
  </Step>
  
  <Step title="Generate code">
    Generate JOOQ Kotlin classes based on the schema into `build/generated-src/jooq/main`
  </Step>
  
  <Step title="Clean up">
    Stop and remove the PostgreSQL container
  </Step>
</Steps>

You should see output similar to:

```
Starting PostgreSQL container...
PostgreSQL container started: jdbc:postgresql://localhost:xxxxx/testdb
Running Flyway migrations...
Flyway migrations completed: 1 migrations executed
Configuring jOOQ to use container: jdbc:postgresql://localhost:xxxxx/testdb

> Task :generateJooq

PostgreSQL container stopped and removed
```

## Step 5: Use the generated code

The generated JOOQ classes are now available in your project. Here's an example of how to use them:

```kotlin
import com.example.jooq.generated.Tables.USERS
import org.jooq.DSLContext
import org.jooq.impl.DSL

fun main() {
    val dsl: DSLContext = DSL.using(connection, SQLDialect.POSTGRES)
    
    // Type-safe query
    val users = dsl
        .selectFrom(USERS)
        .where(USERS.USERNAME.eq("john.doe"))
        .fetch()
    
    println(users)
}
```

<Note>
  The generated code is automatically added to the Kotlin source set, so your IDE will recognize it immediately.
</Note>

## Automatic generation on build

The plugin automatically hooks into the Gradle build lifecycle. The `compileKotlin` task depends on `generateJooq`, so JOOQ code is regenerated whenever you build your project:

```bash
./gradlew build
```

This ensures your JOOQ classes are always up-to-date with your migrations.

## Adding more migrations

As your schema evolves, simply add new migration files:

```text
src/main/resources/db/migration/
  ├── V1__Initial_schema.sql
  ├── V2__Add_orders_table.sql
  └── V3__Add_user_preferences.sql
```

Example of a second migration:

```sql V2__Add_orders_table.sql
CREATE TABLE orders (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    total_amount DECIMAL(10, 2) NOT NULL,
    status VARCHAR(50) NOT NULL,
    created_at TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_orders_user_id ON orders(user_id);
CREATE INDEX idx_orders_status ON orders(status);
```

Run `./gradlew generateJooq` again to update the generated code with the new table.

## Advanced configuration

### Custom PostgreSQL version

If you need a specific PostgreSQL version:

```kotlin build.gradle.kts
jooqConventions {
    postgresVersion = "15-alpine"  // or "14", "13", etc.
}
```

### Multiple schemas

To generate code for a specific schema:

```kotlin build.gradle.kts
jooqConventions {
    inputSchema = "myapp"  // default is "public"
}
```

### Excluding tables

To exclude specific tables from code generation (regex pattern):

```kotlin build.gradle.kts
jooqConventions {
    excludedTables = "flyway_schema_history|temp_.*"  // excludes flyway history and temp tables
}
```

## Troubleshooting

### Docker not running

If you see an error about Docker:

<Warning>
  Ensure Docker is installed and running. Testcontainers requires Docker to create the temporary PostgreSQL database.
</Warning>

### Migration not found

If migrations aren't being applied, verify:

1. Migrations are in `src/main/resources/db/migration/`
2. Files follow the naming convention: `V{version}__{description}.sql`
3. File extensions are `.sql` (not `.txt` or other)

### Generated code not visible in IDE

If your IDE doesn't recognize the generated code:

1. Refresh Gradle: `./gradlew --refresh-dependencies`
2. Rebuild the project: `./gradlew clean build`
3. Reload the Gradle project in your IDE (e.g., in IntelliJ IDEA: Gradle tool window > Reload button)
